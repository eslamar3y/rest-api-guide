01 URL Naming & Structure تسمية الـ Endpoints
0x01 استخدم Nouns مش Verbs في الـ URL Naming

الـ URL يعبر عن الـ Resource. الـ HTTP Method هي الـ Action.

صح
GET /api/users POST /api/users GET /api/users/5 PUT /api/users/5 DELETE /api/users/5
غلط
GET /api/getUsers POST /api/createUser GET /api/getUserById/5 POST /api/updateUser/5 POST /api/deleteUser/5
💡 الـ HTTP Method هي الـ verb — مش المفروض تكررها في الـ URL أبداً.
0x02 الجمع للـ Collections، Nested للـ Relations Naming
صح
GET /api/users // all users GET /api/users/5 // user #5 GET /api/users/5/posts // posts of user #5 POST /api/users/5/posts // create post for user #5
⚠️ ما تعمقش أكتر من مستويين في الـ nesting — يصعب القراءة.
0x03 kebab-case في الـ URL دايماً Naming
صح
/api/user-profiles /api/blog-posts /api/order-items
غلط
/api/userProfiles /api/BlogPosts /api/order_items
0x04 Query Params للـ Filtering, Sorting, Pagination Naming
Query Params
// Filtering GET /api/products?category=phones&status=active // Sorting GET /api/products?sort=price&order=desc // Pagination GET /api/products?page=2&per_page=20 // Search GET /api/products?q=iphone
0x05 HTTP Methods — كل method وليها معنى Naming
GET → Read (safe + idempotent) POST → Create (not idempotent) PUT → Full Update (idempotent) PATCH → Partial Update DELETE → Delete (idempotent) // PUT = send full object: PUT /users/5 → {name, email, phone, address} // PATCH = send only changed fields: PATCH /users/5 → {name: "Ahmed"}
0x06 Actions على Resources — حالات خاصة Naming
صح — actions as sub-resources
POST /api/orders/5/cancel POST /api/users/5/activate POST /api/posts/5/publish POST /api/users/5/send-verification // Relationship actions: POST /api/posts/5/likes // like DELETE /api/posts/5/likes // unlike

03 Authentication & Authorization الحماية والصلاحيات
0x09 JWT vs Sanctum vs Passport Auth
  • Laravel Sanctum — الأفضل للـ SPA وـ Mobile. Simple tokens. يدعم cookie-based للـ SPAs.
  • Laravel Passport — OAuth2 كامل. للـ third-party "Login with our platform".
  • JWT (tymon/jwt-auth) — Stateless. الـ token فيه كل المعلومات بدون DB lookup.
Sanctum
$token = $user->createToken('auth_token') ->plainTextToken; Route::middleware('auth:sanctum')->group(function() { // protected routes here });
0x0A Authorization Header الصحيح Auth
صح
Authorization: Bearer eyJhbGci... // axios: axios.defaults.headers .common['Authorization'] = `Bearer ${token}`
خطر
GET /api/users?token=eyJ... // Token في URL! // بيظهر في server logs // وبراوزر history
🚨 الـ Token في الـ URL بيتسجل في الـ logs — خطر أمني كبير.
0x0B Token Scopes والصلاحيات Auth
Sanctum Scopes
$token = $user->createToken( 'mobile-app', ['read:posts', 'write:comments'] )->plainTextToken; Route::middleware([ 'auth:sanctum', 'abilities:write:comments' ])->post('/comments', ...); $request->user()->tokenCan('write:comments');
0x0C Refresh Tokens Strategy Auth
Response after login
{ "access_token": "eyJ...", // 15min–1hr "refresh_token": "abc...", // 7–30 days "expires_in": 3600, // seconds "token_type": "Bearer" } // Refresh endpoint: POST /api/auth/refresh // body: { refresh_token: "abc..." }

0x0F API Resource في Laravel Design
UserResource.php
class UserResource extends JsonResource { public function toArray($request) { return [ 'id' => $this->id, 'name' => $this->name, 'email' => $this->email, 'created_at' => $this->created_at ->format('Y-m-d'), 'posts_count' => $this->posts_count, // ❌ Never expose: password, token ]; } } return new UserResource($user); // single return UserResource::collection($users); // many
0x10 Global Exception Handler Errors
Handler.php
public function render($request, Throwable $e) { if ($request->expectsJson()) { if ($e instanceof ModelNotFoundException) return response()->json([ 'error' => 'NOT_FOUND', 'message' => 'Resource not found' ], 404); if ($e instanceof AuthenticationException) return response()->json([ 'error' => 'UNAUTHENTICATED', 'message' => 'Please login first' ], 401); } return parent::render($request, $e); }
05 API Versioning إدارة إصدارات الـ API
0x11 طرق الـ Versioning — المقارنة Versioning
URL Versioning — الأفضل
GET /api/v1/users // clear, debuggable GET /api/v2/users // breaking changes here
Header — صعب testing
Accept: application/vnd.api+json;version=2
Query Param — مش clean
GET /api/users?version=1
💡 URL Versioning — واضح، يشتغل مع كل clients، سهل الـ debugging.
0x12 Versioning في Laravel Versioning
routes/api.php
Route::prefix('v1') ->namespace('App\Http\Controllers\Api\V1') ->group(function() { Route::apiResource('users', UserController::class); }); Route::prefix('v2') ->namespace('App\Http\Controllers\Api\V2') ->group(function() { Route::apiResource('users', UserController::class); }); // Controllers/Api/V1/UserController.php // Controllers/Api/V2/UserController.php
⚠️ لا تحذف الـ old version فجأة — أعطي clients وقت للـ migration.

Paginated Response

0x14 Rate Limiting Performance
RouteServiceProvider
RateLimiter::for('api', fn(Request $r) => Limit::perMinute(60) ->by($r->user()?->id ?: $r->ip()) ); Route::middleware(['throttle:api'])->group(...);
  • Response headers: X-RateLimit-Limit: 60
  • X-RateLimit-Remaining: 45
  • When exceeded → 429 + Retry-After: 30

Cache + Invalidate
public function index() { return Cache::remember( 'products.all', now()->addMinutes(30), fn() => Product::with('category')->get() ); } public function store(Request $req) { $p = Product::create($req->validated()); Cache::forget('products.all'); // invalidate! return new ProductResource($p); }
0x16 Selective Fields — ?fields= Performance
صح
GET /api/users?fields=id,name,email public function index(Request $request) { $query = User::query(); if ($f = $request->input('fields')) { $allowed = ['id','name','email','created_at']; $query->select( array_intersect(explode(',',$f), $allowed) ); } return $query->paginate(20); }
07 Security الحماية والأمان
0x17 HTTPS دايماً — مش اختياري Security
  • كل الـ endpoints على HTTPS — HTTP بيبعت tokens plain text
  • Force HTTPS: URL::forceScheme('https')
  • HSTS: Strict-Transport-Security: max-age=31536000
🚨 HTTPS مش بس للـ login — كل الـ endpoints. الـ tokens بتتسرب على HTTP.
0x18 Input Validation دايماً Security
خطر
User::create( $request->all() ); // DANGEROUS!
صح
public function store( StoreUserRequest $req ) { User::create($req->validated()); }
StoreUserRequest
'name' => ['required', 'string', 'max:100'], 'email' => ['required', 'email', 'unique:users'], 'password' => ['required', 'min:8', 'confirmed'],
0x19 CORS Configuration Security
config/cors.php
return [ 'paths' => ['api/*'], 'allowed_origins' => [ 'https://yourapp.com', 'https://admin.yourapp.com', ], 'allowed_methods' => ['GET','POST','PUT','DELETE'], 'allowed_headers' => ['Content-Type','Authorization'], // ❌ Never: 'allowed_origins' => ['*'] ];
0x1A ما تعرضش معلومات حساسة Security
  • ما ترجعش password أو remember_token أبداً
  • في 500 errors ما ترجعش الـ stack trace في production
  • ما تكشفش الـ database structure في رسائل الـ errors

08 أسئلة الانترفيو REST API · 20 سؤال
Q01إيه هو REST وإيه مبادئه الأساسية؟▶

REST (Representational State Transfer) هو architectural style. مبادئه الست: Stateless كل request كامل بنفسه، Client-Server فصل frontend عن backend، Cacheable، Uniform Interface، Layered System، Code on Demand (اختياري).

Q02إيه الفرق بين 401 و403؟▶

401 Unauthorized = مش عارف إنت مين. مفيش token أو token غلط.

403 Forbidden = عارف إنت مين بس مش مسموحلك تعمل ده.

GET /api/profile (no token) → 401 DELETE /api/admin/users (user role) → 403
Q03إيه الفرق بين PUT وـ PATCH؟▶

PUT = Full replacement. لازم تبعت كل fields.

PATCH = Partial update. بتبعت اللي اتغير بس.

PUT /users/5 → {name, email, phone, address} // all PATCH /users/5 → {name: "Ahmed"} // partial
Q04إيه هو Idempotent وإنهي methods idempotent؟▶

الـ Idempotent operation — لو عملتها أكتر من مرة النتيجة نفسها.

Idempotent: GET, PUT, DELETE, HEAD, OPTIONS

Not Idempotent: POST — كل مرة بتعمل record جديد.

DELETE /users/5 → 204 (deleted) DELETE /users/5 → 404 (already gone) ✓ same outcome POST /orders → order #1 POST /orders → order #2 ✗ different!
Q05إيه الفرق بين Authentication وـ Authorization؟▶

Authentication = "إنت مين؟" — التحقق من الهوية.

Authorization = "مسموحلك بإيه؟" — التحقق من الصلاحيات بعد معرفة الهوية.

Authentication بتيجي الأول دايماً.

Q06إيه هو JWT وإزاي بيشتغل؟▶

JWT — stateless token بـ 3 parts (header.payload.signature).

eyJhbGciOiJIUzI1NiJ9 // Header .eyJ1c2VyX2lkIjo1fQ // Payload: {user_id: 5, exp: ...} .SflKxwRJSMeKKF2QT4fw // Signature

الـ server بيتحقق من الـ signature فقط — ما بيعملش DB lookup في كل request.

Q07إيه هو CORS وليه بيحصل؟▶

Browser security policy بتمنع JavaScript على domain من طلب data من domain تاني. الحل: الـ server يرجع headers بتسمح للـ origins المعينة.

Access-Control-Allow-Origin: https://app.com Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Q08إيه الفرق بين REST وـ GraphQL؟▶

REST: Multiple endpoints. كل endpoint بيرجع structure محدد. ممكن over/under-fetching.

GraphQL: Endpoint واحد. الـ client بيطلب بالظبط الـ data اللي يحتاجها.

// REST: multiple requests GET /users/5 + GET /users/5/posts // GraphQL: one request query { user(id: 5) { name, posts { title } } }
Q09إيه هو Rate Limiting وليه مهم؟▶

بيحدد عدد الـ requests اللي client ممكن يعملها في وقت معين. مهم لمنع DDoS, brute force, API abuse. عند التجاوز → 429 Too Many Requests + Retry-After header.

Q10إيه هو API Versioning وليه مهم؟▶

بيتيح تغيير الـ API بدون كسر الـ clients القديمة. لما تعمل breaking change بتعمل version جديد.

GET /api/v1/users → old format (still works) GET /api/v2/users → new format
Q11إيه هو Stateless في REST؟▶

الـ server ما بيحتفظش بأي state عن الـ client بين الـ requests. كل request لازم يحمل كل المعلومات المحتاجة.

// Stateful (session): POST /login → server saves session → GET /profile reads session // Stateless (token): POST /login → returns token → GET /profile + token
Q12إزاي تتعامل مع Pagination في الـ API؟▶

Offset Pagination: ?page=2&per_page=20 — سهل بس بيبطأ مع كبر البيانات.

Cursor Pagination: ?cursor=eyJ...&per_page=20 — أسرع للـ real-time data.

User::orderBy('id')->cursorPaginate(20);
Q13إيه هو HATEOAS؟▶

مبدأ في REST بيقول إن الـ response لازم يحتوي على links للـ related actions.

{ "id": 5, "name": "Ahmed", "links": { "self": "/api/users/5", "posts": "/api/users/5/posts", "delete": "/api/users/5" } }
Q14إزاي تعمل API Documentation صح؟▶

الـ Docs لازم تكون شاملة وفيها أمثلة حقيقية ومحدثة مع كل تغيير. في Laravel: Swagger/OpenAPI مع darkaonline/l5-swagger أو Postman Collections.

Q15إيه هو Webhook وإزاي بيختلف عن API؟▶

API: أنت بتطلب (pull). Webhook: هو بيبعتلك (push) لما حاجة تحصل.

// API: you ask each time GET /api/payment-status/123 // Webhook: Stripe notifies you POST https://yourapp.com/webhooks/stripe { "event": "payment.completed", "amount": 100 }
Q16إيه هو API Gateway؟▶

Entry point واحد لكل الـ requests. بيعمل: Authentication, Rate Limiting, Load Balancing, Routing للـ microservices, Logging. أمثلة: AWS API Gateway, Kong, Nginx.

Q17إزاي تتعامل مع File Upload في الـ API؟▶
POST /api/users/5/avatar Content-Type: multipart/form-data $request->validate(['avatar' => ['required', 'image', 'max:2048']]); $path = $request->file('avatar')->store('avatars', 'public'); return response()->json(['url' => asset('storage/'.$path)]);
Q18إيه هو Content Negotiation؟▶

بيتيح للـ client يحدد نوع الـ response عن طريق الـ Accept header.

Accept: application/json // want JSON Accept: application/xml // want XML // Laravel: if ($request->wantsJson()) { ... } if ($request->expectsJson()) { ... }
Q19إيه هو Idempotency Key وامتى تستخدمه؟▶

بيمنع تنفيذ نفس الـ operation مرتين (زي الـ payments) لو الـ request اتبعت مرتين بالغلط.

POST /api/payments Idempotency-Key: uuid-unique-per-request if (Cache::has('pay_'.$key)) return Cache::get('pay_'.$key); // process + cache result for 24h
Q20إيه الفرق بين JSON وـ XML في الـ API؟▶

JSON: أخف، أسهل parsing، native في JS. الـ standard للـ modern REST APIs.

XML: أثقل، verbose. يُستخدم في legacy systems وـ SOAP.

JSON: { "name": "Ahmed", "age": 25 } XML: <user><name>Ahmed</name><age>25</age></user>
